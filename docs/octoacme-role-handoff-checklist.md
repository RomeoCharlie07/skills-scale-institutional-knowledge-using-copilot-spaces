## Purpose
Ensure smooth transitions when team members change roles, leave the team, or when responsibilities are reassigned. This checklist helps preserve institutional knowledge and maintain project continuity.

## When to Use
- Team member is leaving the project or organization
- Responsibility is being transferred to another team member
- Role is being reassigned due to restructuring
- Extended leave (parental, medical, sabbatical)

---

## Pre-Handoff Preparation

### Outgoing Team Member
- [ ] Document all active work items and their current status
- [ ] List all ongoing responsibilities and recurring tasks
- [ ] Identify critical deadlines within the next 30/60/90 days
- [ ] Prepare list of key contacts and stakeholder relationships
- [ ] Document access permissions and credentials (to be transferred securely)
- [ ] Update all process documentation to reflect current practices

### Incoming Team Member / Manager
- [ ] Identify the incoming person or interim coverage plan
- [ ] Schedule handoff meetings (recommend 2-3 sessions minimum)
- [ ] Review role responsibilities and expectations
- [ ] Ensure access to required systems and tools

---

## Knowledge Transfer Checklist

### Documentation
- [ ] All project documentation is up to date
- [ ] Process docs are accessible and current
- [ ] Key decisions and their rationale are documented
- [ ] Technical specifications and architecture docs are current
- [ ] Runbooks and operational procedures are documented

### Relationships & Communication
- [ ] Key stakeholders are introduced to the incoming person
- [ ] Recurring meeting invites are transferred
- [ ] Communication channels (Slack, email lists) are updated
- [ ] Escalation paths are communicated

### Active Work
- [ ] All active issues/tickets are reassigned
- [ ] Sprint commitments are reviewed and adjusted
- [ ] Pending reviews or approvals are transferred
- [ ] Outstanding decisions are documented and handed off

### Tools & Access
- [ ] System access is provisioned for incoming person
- [ ] Admin rights and permissions are transferred
- [ ] Shared credentials are securely rotated
- [ ] Tool configurations and preferences are documented

---

## Handoff Meeting Agenda Template

### Session 1: Overview & Context (60 min)
- Role overview and key responsibilities
- Current project status and priorities
- Key stakeholders and relationships
- Access to documentation and systems

### Session 2: Deep Dive (60-90 min)
- Walk through active work items
- Review processes and workflows
- Discuss ongoing challenges and risks
- Shadow critical tasks if possible

### Session 3: Q&A & Final Items (30-60 min)
- Address remaining questions
- Verify access and permissions
- Confirm handoff completion
- Schedule follow-up check-in if needed

---

## Post-Handoff Follow-Up

### Incoming Team Member
- [ ] Complete initial review of all documentation
- [ ] Reach out to key stakeholders to introduce yourself
- [ ] Attend first occurrence of all recurring meetings
- [ ] Document any gaps or questions discovered
- [ ] Schedule follow-up with manager after 2 weeks

### Manager
- [ ] Confirm all access and permissions are working
- [ ] Check in with incoming person after 1 week
- [ ] Ensure stakeholders are aware of the transition
- [ ] Address any knowledge gaps identified
- [ ] Remove access for outgoing team member (if applicable)

---

## Role-Specific Handoff Notes

### Project Manager Handoff
- Current project status and timeline
- Risk register and mitigation plans
- Stakeholder communication cadence
- Upcoming milestones and decisions

### Product Manager Handoff
- Product roadmap and priorities
- Backlog state and upcoming features
- User research and feedback sources
- Success metrics and dashboards

### Developer Handoff
- Codebase overview and architecture
- Local development setup
- Code ownership areas
- Pending technical decisions

### Scrum Master Handoff
- Team rituals and cadence
- Current sprint state
- Team dynamics and concerns
- Improvement action items in progress

### QA Lead Handoff
- Test strategy and coverage
- Known issues and defect backlog
- Test environment setup
- Automation framework details

### Technical Writer Handoff
- Documentation inventory
- Style guides and templates
- Publishing workflows
- Outstanding documentation gaps
